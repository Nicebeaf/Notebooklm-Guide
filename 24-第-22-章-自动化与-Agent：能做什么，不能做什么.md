# 第 22 章 自动化与 Agent：能做什么，不能做什么

"NotebookLM 能自动化吗？能让 AI Agent 操作它吗？能让它定时帮我做研究吗？"

这一章做诚实回答。按稳定性分四层，从"现在就能用"到"目前只是概念"。

## ① 正式可用：NotebookLM 与 Gemini 联动

这是目前唯一稳定的官方跨产品工作流。Gemini App 中的对话记录可以作为来源导入 NotebookLM。

**实际可以做的事**：用 Gemini 做初筛和快速搜索，"最近三个月关于 XX 行业的重要动态"，快速浏览 Gemini 的回答，把有价值的结果链接导入 NotebookLM，做深度研究和归档。

**实际不能做的事**：这不是"自动化"。每次都需要你手动操作，在 Gemini 中对话，手动选择导入，手动审核导入的内容。它是手动的跨产品接力，不是自动的后台流水线。

## ② 预览阶段：Enterprise Admin API

面向企业 Google Workspace 管理员的 Notebook 管理 API。截至本书写作时（2026 年 7 月），功能仍为预览阶段。

**能做什么**：企业管理员可以通过 API 管理组织内的 Notebook 的创建、权限、数据治理。

**不适用于什么**：个人用户无法使用。这不是面向普通用户的自动化接口。不保证 API 的稳定性和长期可用性。

## ③ 实验性质：社区 MCP

社区中存在非官方的 MCP（Model Context Protocol）接口实现，通过浏览器自动化操作 NotebookLM 的网页界面。

**风险清单**：
- **非官方接口**：随时可能因 NotebookLM 产品更新而失效。Google 没有承诺维护社区 MCP 的兼容性。
- **凭据风险**：需要你的 Google 账号登录态。任何以你的身份操作 NotebookLM 的第三方接口，都涉及凭据安全的考量。
- **维护成本**：需要持续跟进接口变化和兼容性修复。如果你的自动化在关键时刻失效了，那天你可能正好需要它。
- **违反服务条款的风险**：自动化操作网页界面可能违反 Google 的服务条款。使用前阅读并理解相关条款。

**建议**：除非你有明确的自动化需求**并且**接受接口随时失效的风险**并且**愿意承担维护成本和合规风险，否则不值得投入。90% 的"自动导入 + 自动提问 + 自动出报告"需求可以通过手动几步完成，多花的时间远少于折腾一个不稳定的自动化。

## ④ 不依赖具体协议的稳定方法

最可靠的"自动化"方法不依赖 NotebookLM 提供 API。它依赖的是**你把 NotebookLM 当作研究层的稳定工作流**，和 Agent 明确的分工边界。

**架构**：
- **事实层（NotebookLM）**：你积累的经过核验的知识，第 7 章标准。这是整个系统的信任基础。
- **执行层（Agent）**：承担执行性任务，搜索新信息、格式化输出、定时提醒。Agent 不替代 NotebookLM 的研究层，Agent 在 NotebookLM 的下游工作。
- **确认层（你）**：每次 Agent 执行后，高风险输出必须经过人工确认。Agent 可以搜索和整理，但最终进 NotebookLM 的来源必须经过你的审核。Agent 可以格式化，但最终发布的内容必须经过你的检查。

**这个架构不依赖 NotebookLM 提供 API。** 它不"自动化 NotebookLM"。它区分了什么适合自动化（搜索、格式化、提醒）和什么不适合自动化（来源审核、事实核验、最终判断）。这种分工在产品迭代中不会过时，因为它基于能力的本质差异，不是基于某个特定接口的存在。

## 自动化能做什么、不能做什么：一张对照表

| 任务 | 能自动化吗 | 原因 |
|------|----------|------|
| 定时搜索特定关键词并返回 URL 列表 | ✅ 能 | 这是搜索层的事，不涉及 NotebookLM |
| 自动把搜索结果导入 NotebookLM | ⚠️ 技术上可在审核后手动完成 | 来源审核不应自动化，第 6 章和第 10 章的批量导入教训 |
| 自动在 NotebookLM 里提问并获取回答 | ⚠️ 技术上可通过社区 MCP，风险见③ | AI 的回答需要你来判断，自动获取≠自动理解 |
| 从 NotebookLM 的回答中自动提取数据并填入表格 | ❌ 高风险 | AI 可能误解数字、混淆口径，第 7 章的核验标准 |
| 定期提醒你该做季度来源清扫了 | ✅ 能 | 这是提醒，不是自动化操作 |

## 验收清单

- ☐ 理解了 Gemini 联动是"手动接力"而非"自动化"
- ☐ 如果考虑使用社区 MCP，已评估凭据风险和失效风险
- ☐ 自动化任务的边界清晰：搜索/格式化/提醒→可自动化；来源审核/事实核验→不可自动化
