# 第 11 章 读论文与文献综述

> 核验状态（2026-07-10）：本章论文书目信息（arXiv 编号、会议、年份）经学术数据库交叉核对；正文不引用论文内部的具体实验数值，理由见第五节。功能操作对照官方帮助中心。
> 贯穿案例：LLM 幻觉检测与缓解研究（2022-2025）

你打开 Zotero，里面有 87 篇关于 LLM 幻觉的论文。导师说"文献综述不要罗列，要有 synthesis"。你盯着屏幕，你记得大部分论文讲了什么，但你不确定它们之间的真正关系。哪几篇在互相反驳？哪几篇用了同一套方法但得出不同结论？哪篇的核心发现已经被后来者推翻了？

**论文文献综述最怕两件事：一是写成流水账（"A 提出……B 提出……C 提出……"），二是漏了关键工作被审稿人抓出来。**

本章教你用 NotebookLM 读论文和做文献综述。贯穿案例是 LLM 幻觉检测与缓解研究，这个领域在 2022-2025 年间经历了爆发式增长，定义不统一、评测碎片化、方法路线分散，是文献综述方法的绝佳示范对象。

本章交付一份**文献综述档案**，按四级证据强度组织的论文地图，附带每篇论文的核心贡献、方法局限和论文之间的真实关系。**不教**：如何判断论文本身的学术质量（那是学术训练的事），如何做原创研究（那是你自己的事）。

## 一、先想问题，再搜论文

**主问题**：LLM 幻觉问题在 2022-2025 年间的研究进展如何？检测方法和缓解策略各自的有效性边界在哪？什么还没被解决？

**子问题清单：**

1. "幻觉"的定义有多少种？不同定义导致什么后果，是否让不同论文报告的"幻觉率"在概念上不可比？
2. 幻觉检测方法分几类？每类的核心假设是什么？在什么条件下失效？
3. 缓解策略分几类？训练阶段、推理阶段、后处理阶段的各自 trade-off 是什么？
4. 各论文的实验设置在哪些关键维度上不同（评测数据、模型、幻觉类型的操作定义）？
5. 这个领域最被引用的结果中，哪些已经在后续工作中被质疑或修正？
6. 五个最重要的开放问题是什么？

**主问题措辞的自我修正**：第一稿写的是"当前最好的幻觉检测方法是什么"，"最好"的判断需要跨论文的标准化基准，而这个领域缺乏统一定义和统一评测框架。改成"检测方法的有效性边界在哪"，这个问题可以用现有文献回答：你不需要知道哪个"最好"，你需要知道每个方法在什么条件下有效、什么条件下失效。

这个修正本身就是文献综述的核心思维：**研究问题必须与证据的可获得性匹配。** 你问了一个文献中无法回答的问题，要么改问题，要么在综述中明确标注"这个问题目前缺乏足够的文献支撑"。

## 二、来源包：三层结构

搜集顺序：**定义问题 → 初筛候选 → 人工逐篇审核 → 对证据不足的子问题定向 Deep Research → 审核后导入。** Deep Research 是补枪，不是起手式。先写清楚问题再搜论文，否则你会被 87 篇论文淹掉。

## 三、建立 Notebook

命名：`论文-LLM幻觉研究-文献综述`。

**导入注意事项**：
- arXiv 论文通过 URL 导入，NotebookLM 抓取全文文本。
- S3-S6 论文中公式密集的段落，追加一条 Note 标注"公式(3)的核心语义是……"，帮助 AI 在引用时正确定位到公式所在的段落并理解其含义。
- S9（审稿意见）作为 Note 手动导入，摘录每条审稿意见中的核心批评点。

**自定义指令**：

```
你是文献综述助手。原则：

1. 引用标注作者+年份+来源编号。跨论文比较时核对实验设置（数据集、模型、幻觉定义）。
   实验设置不同 = 数字不直接可比。
2. 术语冲突时做对照："X 论文的术语 Y ≈ Z 论文的术语 W（区别在于……）"。
3. 识别每篇论文的"声明范围"（声称解决什么）和"实际验证范围"（实验覆盖了什么）。
   两者差距越大，结论越需要谨慎引用。
```

## 四、提示链

以下六步按序执行。每一步的输出是下一步的输入。更多可直接复制的提示语见附录 D。

**P1 · 定义与分类对照**：「列出所有来源中关于 LLM 幻觉的定义和分类体系。当不同论文对同一个概念使用不同术语时，做对照说明：谁≈谁，区别在哪，这种术语差异会不会导致误解。」

设计意图：文献综述的第一个陷阱，每个作者都用自己定义的术语。不做术语对照，后面的所有比较都是在不同定义之间空转。这一步是整个综述的概念地基。

**P2 · 方法图谱**：「将检测和缓解方法分别聚类到技术路线下。每条路线：核心思想一句话、代表性论文、在什么基准上取得什么效果、已知的失效条件。」

设计意图：读 20 篇论文不是为了记住 20 个标题，是看到背后的 5-6 条技术路线的分布。聚类降低了记忆负担，提升了理解的体系性。

**P3 · 实验设置差异**：「提取各论文的实验设置关键维度：评测数据集、评测的 LLM 型号和规模、幻觉的操作定义、评估指标。明确标注哪些论文的数字因为设置不同而不当并列。」

设计意图：文献综述最隐蔽的错误不是漏了某篇论文，是**把不同实验设置下的数字摆在同一个表格里让它们看起来可比**。这一步逼模型暴露不可比性。第五节关键数字表就是这个提示的产出。

**P4 · 争议与修正**：「找出被后来研究挑战的核心结论。至少三个例子，每个标注原始论文的发现、后来质疑的论文及依据、当前领域共识的强度（'轻微修正'/'方法争议'/'结论可能不成立'）。」

**P5 · 开放问题**：「列出本领域 5 个最重要的未解决问题。每个附带：为什么难解决、目前最近的工作是什么、可能的突破口在哪。」

**P6 · 综述档案生成**：「将 P1-P5 的分析整合为文献综述档案，按四级证据强度组织。」

操作路径：将 P1-P5 的回答 Save to Note → Convert to source → 在来源面板只勾选转换后的新来源 → 在 Chat 中执行 P6 → 将 P6 的回答再次 Save to Note 并 Convert to source → 只勾选最终来源 → 生成 Mind Map。这样 Mind Map 反映的是你核验和分析过的结构化综述，而非全部原始论文的自动摘要。

**P2-P4 的共同点**：逼模型暴露矛盾和局限。"不要调和它们"是整条提示链里最重要的半句话。

## 五、关键数字表：实验设置差异的现场教学

这张表展示的不是论文的实验数据，是**同一个领域中，因为实验设置不同，数字为什么不能直接并列**。

| 维度 | FActScore (S3) | SelfCheckGPT (S4) | 不可直接对比的原因 |
|------|---------------|-------------------|------------------|
| 评测任务 | 传记生成（Wiki 风格人物描述） | WikiBio + 合成问答 | 任务类型不同，一个评测事实性传记，一个评测 QA 幻觉 |
| 幻觉定义 | 原子事实与外部知识库的一致性 | 多次采样输出之间的自洽性 | 一个测"对外部世界是否一致"，一个测"对自己的多次输出是否一致"，概念上不是同一个东西 |
| 衡量指标 | FactScore（句子级原子事实精确率） | AUC（检测器区分有/无幻觉的能力） | 一个是**模型错误率**，一个是**检测器性能**，度量对象完全不同 |
| 错误并列示例 | 「模型错误率」与「检测器判别力」两类数值 | — | 并列会产生"32%的幻觉能被 86% AUC 的检测器抓到"的错误解读 |

**教学点**：文献综述最隐蔽的错误不是漏了某篇论文，而是**把不同实验设置下的数字摆在同一个表格里让它们看起来可比**。凡并列不同论文的数字，先检查它们的度量对象在概念上是不是同一个东西。不是同一个东西的，不进同一个表格。

## 六、常见陷阱

**陷阱一：不同论文的数字不当并列。** 如第五节关键词表示，FActScore 的"事实错误率"和 SelfCheckGPT 的"检测 AUC"度量的是完全不同的东西。把不同基准、不同任务、不同操作定义下的数字放在同一个表格里产生"可比"的错觉：这是文献综述中最隐蔽也最常见的错误。

**陷阱二：盲目信任论文的"声明范围"。** 一篇论文声称"我们解决了 XX 问题"，但实验可能只覆盖了特定模型、特定数据集、特定幻觉类型。始终追问：这个结论在什么条件下成立、在什么条件下可能失效？声明范围和验证范围之间的差距，是判断论文结论可靠性的核心维度。

## 七、最终成品：LLM 幻觉研究文献综述档案（摘要）

**分析框架声明**：本章将幻觉研究分为检测（detection）与缓解（mitigation）两个阶段；检测按技术路线分"自洽性""外部知识""内部表示""不确定性估计"四类，缓解按干预阶段分"训练""推理""后处理"三类。这是一个工作分类框架，不是领域内唯一的分类法。

### A. 已确认事实（一手出处，多来源共识）
- **A1**：LLM 幻觉是多维度现象，社区共识区分"事实性幻觉"（内容与外部世界不一致）与"忠实性幻觉"（内容与输入上下文不一致）。来源：S1 和 S2 共识。
- **A2**：自洽性检测方法（如 SelfCheckGPT）是当前最实用的黑盒方案，前提是模型在多次采样中表现不一致。对"模型自信地错"的场景失效。来源：S4 + S2 后续分析。
- **A3**：没有单一缓解方法覆盖所有幻觉类型。训练干预（RLHF）、推理干预（解码策略）和 RAG 互补。来源：S2 综合分析。

### B. 多来源支持的趋势
- **B1**：领域从"检测幻觉"转向"幻觉归因"，不仅要发现错了，还要知道"为什么错"。S2 称这是 2024 年的核心转向。
- **B2**：评测生态碎片化（30+ 基准、不同定义、不同指标）是阻碍跨论文比较的最大结构性问题。

### C. 作者工作假设
- **C1**：当评测生态统一时，当前大量被引的结果可能需要重新评估，在不同的操作定义下，结论可能不迁移。

### D. 待验证问题
- **D1**：自信幻觉的检测，自洽性方法对此失效，目前无公认有效方案。
- **D2**：缓解方法的成本-收益 Pareto 前沿，缺乏系统性的成本分析。
- **D3**：跨模型架构（非 Llama 系列）的泛化性验证。
