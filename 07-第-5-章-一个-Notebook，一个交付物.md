# 第 5 章 一个 Notebook，一个交付物

你新建了一个 Notebook，兴奋地把各种相关材料拖进去，行业报告、竞品分析、读书笔记、会议记录。三个月后，这个 Notebook 里有 47 个来源，涵盖四个不同的话题方向。你问它一个问题，回答里混杂了竞品数据和读书摘录。你发现你分不清哪个引用来自哪类信息。

**Notebook 不是越大越好。** 一个没有清晰边界的 Notebook 会变成信息沼泽，不是不能用了，是每次用完你都要花额外的时间确认"这个结论是基于什么上下文"。

## 核心原则：一个 Notebook，一个交付物

这句话是全书 Notebook 组织法的基石。

**"一个 Notebook，一个交付物"意味着**：你创建的每个 Notebook，都对应一个明确的**问题、决策或产出**。不是"和 AI 相关的所有东西"，而是"我需要为下周的 AI 战略会议产出一份行业分析"。不是"我关注的所有公司"，而是"我要评估 X 公司值不值得投资"。

对应关系的价值在于：**当你打开一个 Notebook 时，你立刻知道它是为什么而存在的，以及它里面的所有来源都服务于同一个目的。**

## 主存储 + 工作台的两层模型

回到第 2 章的类比：NotebookLM 是你的工作台，不是你的图书馆。

- **主存储**：Google Drive、本地文件系统、文献管理器（Zotero / Paperpile）、云盘：这是你的"图书馆"。所有原始文件在这里做长期保存、版本管理、去重和备份。
- **NotebookLM（工作台）**：每次启动一个研究、决策或交付任务时，从主存储中取需要的材料，导入新建的 Notebook。加工、综合、产出后，成品归档回主存储。

这套两层模型解决了"把 NotebookLM 当存储用会撞上限"的问题，也解决了"所有东西混在一起信噪比暴跌"的问题。你不需要把主存储里所有的 AI 行业报告都放进 NotebookLM，你只需要放和当前任务相关的那 10 份。

## 三种组织模式及其边界

"一个交付物"在实践中表现为三种模式。选择哪种取决于你的任务性质。

### 模式一：项目型 Notebook

**适用场景**：有明确起止时间和交付物的具体项目。

| 项目 | Notebook 名称 |
|------|-------------|
| 写这本书 | `书-NotebookLM实战指南-v2` |
| 准备一个客户提案 | `项目-XX客户提案-2026Q3` |
| 一次求职 | `求职-产品总监-2026` |
| 准备一个演讲 | `演讲-AI Agent趋势-2026-07` |

项目型 Notebook 的特征：有生命周期，项目结束，Notebook 归档。有明确的所有者，你会知道"这个 Notebook 是用来产出什么的"。

### 模式二：行业/主题研究 Notebook

**适用场景**：对一个行业或主题做持续跟踪和深度研究。

可以复用旧第 6 章的分层结构：基石信息来源（官方统计、龙头年报）→ 深度分析来源 → 动态跟踪 → 个人分析笔记。但关键是，**这个 Notebook 仍然服务于一个明确的问题**。它不是"关于储能行业的所有东西"，而是"回答'储能行业当前竞争格局和投资机会'这个问题的研究 Notebook"。

如果同一个行业你换了一个问题，比如从"竞争格局"变成"技术路线对比"，考虑新建一个 Notebook，而不是在旧的上面追加。

### 模式三：个人积累型 Notebook

**适用场景**：没有明确的截止日期，但有一个持续聚焦的主题。比如素材库、观点库。

这种模式在第 17-18 章（内容知识库和选题库）中展开。这里只强调一条：**即使是积累型的 Notebook，也有明确的边界**，"写作素材"和"A 行业研究"不会混在同一个 Notebook 里。

## 三种模式的边界

| | 项目型 | 行业研究型 | 积累型 |
|---|---|---|---|
| 驱动力 | 明确交付物 | 明确问题 | 持续关注 |
| 生命周期 | 有结束 | 持续，但问题可更新 | 长期 |
| 边界 | 项目范围 | 问题和主题 | 素材类型 |
| 标签核心 | 来源类型 | 证据层级 | 素材状态 |

不要在同一个 Notebook 里混合三种模式。项目型 Notebook 里塞进积累型素材 = 回答被无关内容稀释。积累型 Notebook 里混入具体的项目来源 = 信噪比暴跌。

## 命名规范

当你有 30 个 Notebook 时，不规范命名 = 每次都要点开才知道里面是什么。

格式：`[类型前缀]-[具体名称]-[阶段/时间]`

| 类型前缀 | 用途 |
|---------|------|
| `项目-` | 有起止时间的交付任务 |
| `研究-` | 行业/主题深度研究 |
| `书-` | 写书/写长篇报告 |
| `学-` | 学习某门课程或技能 |
| `个人-` | 持续积累的个人素材 |
| `归档-` | 已完成、不再活跃更新 |
